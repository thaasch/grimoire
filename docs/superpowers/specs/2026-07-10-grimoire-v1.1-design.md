# GRIMOIRE v1.1 — Variation Pads, QoL, Hardening: Design

**Date:** 2026-07-10
**Status:** Approved by user (brainstorming session)
**Baseline:** v1 on `main` (commit ccc10fd), spec `2026-07-10-pnp-soundboard-design.md`

## Overview

Second iteration of GRIMOIRE. Three strands:

1. **Random-variation pads** — one pad backed by several one-shot samples; each trigger
   plays a random member (never the same twice in a row).
2. **QoL pack** — scene duplication, configurable fade durations, preset pad colors.
3. **Hardening** — the follow-ups agreed at v1 review: debounced volume persistence,
   persistence-error toasts, accessibility pass, engine/store test gaps.

Deferred to later cycles (own specs): v1.2 music playlist/rotation, v1.3 waveform trim
editor.

## Non-goals

- No playlist lane, no waveform rendering or trimming (later cycles).
- No mixed-type sets: variation sets contain **one-shots only**.
- No per-scene fade overrides — fade settings are global.
- No free-color picker — colors come from a fixed 8-token palette.

## 1. Variation sets

### Data model

```ts
export interface VariationSet {
  id: string;            // uuid, same id space as Sound.id (collision-free)
  name: string;
  emoji: string;         // default '🎲'
  soundIds: string[];    // members, invariant: length >= 2, one-shots only
  defaultVolume: number; // 0..1
  color?: PadColor;      // see §3 Pad colors
}
```

- `Pad.soundId` is unchanged and may hold a **sound id or a set id**. A stores helper
  resolves it:

```ts
type Ref =
  | { kind: 'sound'; sound: Sound }
  | { kind: 'set'; set: VariationSet }
  | null;
export function resolveRef(id: string): Ref;
```

- New writable store `sets: Writable<VariationSet[]>`; new IndexedDB object store
  `sets` (key: id) — **DB version 1 → 2**, upgrade only adds the store.
- Zip manifest: optional `sets: VariationSet[]` field, manifest `version` stays `1`
  (additive; v1 zips import unchanged, v1 apps importing v1.1 zips show orphan pads for
  set references — accepted).
- Export includes sets; import merges sets by id like sounds/scenes.

### Playback semantics

- `engine.play(sound, volume?, fadeIn?, instanceKey = sound.id)` — new optional
  parameter. The instance's `soundId` is `instanceKey`; the buffer lookup stays keyed by
  the actual `sound.id`. Pads, `isSoundPlaying`, `stopSound`, and mixer chips therefore
  work unchanged, keyed on whatever the pad references.
- `triggerRef(id, volume?)` — the new entry point; `triggerPad` routes through it, and
  `triggerSound` remains as the sound-only path (library preview buttons): resolve the
  id; for a set, pick a random member ≠ the previous
  pick for that set (runtime `Map<string, string>`, not persisted; `Math.random`,
  injectable for tests). Volume: pad override → set.defaultVolume.
- Missing member audio: try another (unpicked) member; only when **no** member is
  playable, `markBroken(setId)`.
- Mixer chip for a set instance shows the set's emoji + name.

### Lifecycle rules

- Deleting a member sound removes its id from every set (cascade).
- A set dropping to 1 member **dissolves**: every pad referencing the set is rewritten
  to the remaining sound id, then the set record is deleted.
- Deleting a set (library ✕): pads referencing it are removed from scenes (same cascade
  as deleting a sound); member sounds stay in the library.
- Adding a loop to a set is rejected with toast `toast.loopInSet` (both languages).

### Creation & management UX

Both creation gestures are **edit-mode only**:

1. **File-drop onto a pad**: dropped audio files upload as normal sounds; if the pad
   references a sound, a set is created from `[pad's sound, ...uploaded]` inheriting
   name/emoji/color/defaultVolume from the pad's sound, and the pad is repointed to the
   set. If the pad already references a set, the uploads join it. Files that classify
   as loops (>30 s) are rejected from the set with `toast.loopInSet` but stay in the
   library as ordinary sounds.
2. **Pad-onto-pad drag**: dragging pad A onto pad B's face (both resolving to
   one-shots) merges: B becomes/extends a set, A is removed from the scene; B's pad
   settings (volume, position) win, A's are discarded. Dropping
   between cells still reorders; the hovered pad face shows a gold merge highlight to
   distinguish the two targets. If either side resolves to a loop, the drop is a no-op
   with `toast.loopInSet`.

Identity: pads and library rows referencing a set show `🎲 <memberCount>` where loops
show ∞.

**SoundEditModal set mode** (when `editingSound` holds a set id): fields name, emoji,
color, volume; a member list, one row per member (name, preview-play button, remove ✕);
no type selector. Removing down to 1 member triggers dissolution after a browser
`confirm()` using key `sets.dissolveConfirm` ('Set auflösen? Der letzte Sound bleibt
als normales Pad.' / 'Dissolve the set? The last sound remains as a normal pad.').

The library lists sets as entries alongside their member sounds (members are not
hidden). "Zur Szene" adds a pad referencing the set. Search matches set names.

## 2. QoL pack

### Scene duplication

Scene tab context menu gains "Duplizieren"/"Duplicate" (key `scenes.duplicate`):
deep-copies pads (volume, autoplay, position), names the copy via key
`scenes.copyName` = `{name} (Kopie)` / `{name} (copy)`, inserts directly after the
original (positions renumbered), activates the copy (no crossfade needed — copies start
silent; normal `activateScene` rules apply).

### Fade settings

```ts
interface Settings {
  // ...existing...
  fades: { stop: number; stopAll: number; crossfade: number }; // seconds
}
// defaults: { stop: 0.3, stopAll: 1.5, crossfade: 1.5 }
```

- New ⚙ button in the top bar (right group) opens a small popover with three labeled
  sliders, range 0–5 s, step 0.1, live value shown. Keys: `settings.title`,
  `settings.fadeStop`, `settings.fadeStopAll`, `settings.fadeCrossfade`.
- All call sites read from settings (`InstanceChip` ✕, Stop All button, Escape handler,
  `activateScene`, scene-switch hotkeys). Engine keeps its defaults as fallback.
- Loading stored settings without `fades` merges defaults
  (`{ ...DEFAULT_SETTINGS, ...stored, fades: { ...DEFAULT_SETTINGS.fades, ...stored.fades } }`).
- Popover a11y and Escape behavior follow the modal rules in §3.

### Pad colors

```ts
export type PadColor =
  | 'gold' | 'violet' | 'blue' | 'green' | 'teal' | 'rose' | 'amber' | 'slate';
```

- `color?: PadColor` on **Sound and VariationSet** (not on Pad) — a sound keeps its
  color in every scene. Default (absent) = `gold` = exactly today's look.
- 8 CSS token pairs in `app.css`: `--accent-<name>` (border/text tint) and
  `--accent-<name>-faint` (background tint), each color-checked against `--bg` for
  contrast. The accent colors the idle pad border, emoji chip, and a small dot in
  library rows. **Playing state remains ember for every color.**
- Picker: a swatch row in SoundEditModal (both modes), swatches are buttons with
  translated `aria-label`s (`color.gold` … `color.slate`).

## 3. Hardening

### Debounced persistence

- `src/lib/persist.ts`: `debouncedPersist(key: string, write: () => Promise<void>, delay = 300)`
  — trailing debounce per key; `flushPersist()` runs all pending writes immediately.
- Applied to: master volume, fade sliders, per-pad volume, sound/set defaultVolume.
  Structural mutations (add/remove/reorder/rename) stay immediate.
- `flushPersist()` wired to `beforeunload` and `visibilitychange→hidden`.

### Persistence-error toasts

- `persist.ts` also exports `guard<T>(p: Promise<T>): Promise<T | undefined>`: awaits,
  catches, toasts `toast.quota` for `QuotaExceededError`, else new key
  `toast.saveFailed` ('Speichern fehlgeschlagen — Änderung ggf. nicht dauerhaft' /
  'Saving failed — change may not persist'), returns undefined on failure.
- Wraps every optimistic IndexedDB write in stores.ts outside the already-guarded
  upload path (`patchScene`, `patchSettings` non-debounced path, `updateSound`,
  scene/sound/set deletion, moveScene).

### Accessibility pass

- **Modals** (LibraryModal, SoundEditModal, new settings popover): `aria-modal="true"`,
  `aria-labelledby` → heading id, focus moves to first focusable on open, Tab/Shift+Tab
  trapped inside, focus returns to the opening control on close.
- **Scene context menu**: ArrowUp/ArrowDown move focus through menu items (roving
  focus), Home/End jump, focus returns to the triggering tab on close.
- **Icon-only buttons** get translated `aria-label`s (✎ edit mode, + new scene, ⚙
  buttons, ✕ close/stop/remove, 🗑 delete, preview-play emoji buttons): keys
  `a11y.editMode`, `a11y.newScene`, `a11y.settings`, `a11y.editSound`, `a11y.remove`,
  `a11y.delete`, `a11y.stop`, `a11y.preview` (reuse `a11y.close` where it fits).
- **Search inputs** (top bar, library) get `aria-label={$t('search.placeholder')}`.
- svelte-check must remain 0 errors / 0 warnings.

### Test gaps to close

- Engine: `decode` (copies input, returns buffer), `hasBuffer`/`now`,
  `setMasterVolume` after context creation (param event), `source.stop` throwing
  (mock made to throw once — instance still marked stopping and removed on ended),
  `play` with explicit `instanceKey`.
- Stores: `duration === 30` boundary (one-shot), set trigger random-no-repeat
  (injected RNG), all-members-missing → set marked broken, member-delete cascade &
  dissolution rewrite, merge-on-drop store action, scene duplication (deep copy +
  renumber + activate), fades merge-on-load, debounce (fake timers: single write per
  burst, flush writes immediately).

## Architecture notes

- New files: `src/lib/persist.ts`, `src/components/SettingsPopover.svelte`, plus a
  shared `src/lib/focus.ts` (focus-trap action used by both modals and the popover).
- `stores.ts` gains set actions (`createSetFromPad`, `addToSet`, `removeFromSet` with
  dissolution, `deleteSet`) — if the file grows past ~450 lines, split set logic into
  `src/lib/sets.ts` (stores re-export for component ergonomics).
- Dependency rules unchanged: components → stores → (db, engine, i18n); engine stays
  Svelte-free apart from its own stores; db knows no audio.
- Migration safety: DB upgrade 1→2 additive; settings fades merged at load; manifest
  additive. A v1.1 export imported into v1.1 round-trips sets losslessly.

## Testing philosophy

As v1: TDD for lib modules (RED→GREEN evidence), UI verified by running the app,
`npm test` + `svelte-check` + build green per task. New UI behaviors that reduce to
store logic (merge, dissolution, duplication) are tested at the store layer.
